# Kiro Hackathon

## General Instructions

### Your Starter Prompt

Type this into Kiro to begin:

> "Before designing, ask me clarifying questions about the solution. Create an architecture diagram showing what this would look like on AWS and why a particular service was chosen. Do not deploy to the cloud, just host the prototype locally. Do not include authentication in the prototype. Use mock data where needed."

---

## Kiro Feature Cheat Sheet

| Feature | When to use it | How |
|---|---|---|
| **Spec-driven mode** | At the START. Before writing any code. | "Create a spec for this feature" — iterate on requirements before Kiro codes |
| **Kiro Powers** | Install right away when you start | Click the ghost+lightning icon on the left sidebar, search and add: <br> • Build full-stack apps with AWS Amplify <br> • Build AWS infrastructure with CDK and CloudFormation |
| **MCP Servers** | When you need Kiro to access AWS docs and best practices | Ask Kiro to install: <br> • AWS Documentation MCP Server <br> • AWS Knowledge MCP Server |

---

## Speed Tips

- Ask Kiro to build the architecture diagram **FIRST** (great for your final presentation)
- If stuck on an error, paste the full error and ask "explain this and fix it"
- Focus on **3-4 core features**, not everything

---

## General Rules

- Deploy locally only (no cloud)
- No authentication in the prototype
- Use mock/dummy data where needed
- Architecture diagram should show what it **WOULD** look like on AWS
